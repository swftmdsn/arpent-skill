# Context Summary

## Trigger

Use only when the user explicitly asks to build or refresh intelligent context
summaries. `arpent index` never triggers this skill automatically.

## Input

- An initialized and indexed Arpent vault.
- Optional path or kind scope from the user.
- Entries returned by `arpent context pending`.

## Steps

1. Run `arpent index` to refresh deterministic inventory and hashes.
2. List work with `arpent context pending --json`, optionally adding
   `--path <relative-path>` or `--kind <folder|note|text>`.
3. Skip every entry whose L1 status is already `fresh`.
4. Load only the required L2 source with
   `arpent context show <path> --level l2`. For folders, start from child L0/L1
   context rather than loading every descendant in full.
5. Produce a factual, standalone summary focused on purpose, key information,
   current state, and useful relationships. Prefer 2-5 sentences and stay below
   180 words.
6. Store it with `arpent context set <path> --source-hash <hash-from-pending> --stdin --provider <agent-or-model-id>`. If the hash no longer matches, discard the summary, reload L2, and regenerate it.
7. Confirm that the path no longer appears in `arpent context pending`.

## Output

Fresh L1 entries in `06_indexes/context_index.json`, each tied to the exact
semantic source hash it summarizes.

## Method

- L0 is deterministic orientation and is regenerated by `arpent index`.
- L1 is optional AI output and is never generated implicitly.
- L2 is the source or child listing loaded only on demand.
- Never regenerate a fresh L1 whose source hash still matches.
- A stale L1 may be read for comparison but must not be presented as current.
- Never summarize binary files or infer facts absent from the source.
- Follow the Arpent skill's primary/adaptive language settings for summaries.
