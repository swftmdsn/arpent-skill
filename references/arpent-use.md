# arpent-use

*Orchestration layer for an agent whose primary surface is an Arpent vault. Sits above the skill-root `SKILL.md` (the mechanics) - load that before acting on the vault.*

**What it is.** A filesystem-native life OS: a clean knowledge base (7 buckets, deterministic routing), a human-owned `me.md` orientation file, plus a delegated, modular memory system beside it. The files are the source of truth; you're the renewable agent operating them.

**When to use it - almost always.** Any time information should be kept, structured, or recalled: captures, lookups, things to remember, reading worth keeping, work on a project or area. Skip only for ephemeral chit-chat or a one-off computation. When unsure, use it.

**Three things to keep in mind:**
1. **Everything has a deterministic place** - route, don't guess; never delete (archive); announce moves; park ambiguity in `00_inbox/unsure/`.
2. **Knowledge → the vault (kept clean); memory → the delegated system** - never dump facts into the vault.
3. **You're renting; the user owns** - read `me.md` early, keep files legible for the next agent, and defer to the user on identity and meaning.
